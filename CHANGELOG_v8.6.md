# v8.6 更新日志

## 发布日期：2026-06-23

## 新增功能

### 💤 随机间断功能

运行过程中随机暂停休息，模拟真实用户的使用习惯，进一步增强防检测能力。

#### 核心特性

1. **智能休息安排**
   - 每隔 10-20 分钟随机安排一次休息
   - 休息时长可自定义（默认 2-5 分钟）
   - 完全随机化，避免规律性被检测

2. **用户界面改进**
   - 在"运行模式"区域新增"随机间断配置"
   - 可勾选启用/禁用
   - 可自定义最小和最大休息时长
   - 提示文字："(模拟真人休息)"

3. **运行时体验**
   - 休息时显示倒计时："💤 休息中: 3:45"
   - 详细的日志输出：
     - 安排休息："💤 已安排下次休息：约 15 分钟后"
     - 开始休息："💤 开始休息 4 分钟（模拟真人行为）"
     - 结束休息："✨ 休息结束，继续浏览"

4. **时间计算优化**
   - 休息时间**不计入**总运行时间
   - 设置 60 分钟，实际工作 60 分钟（休息额外）
   - 更符合真实用户行为

#### 使用建议

| 运行时长 | 建议配置 |
|---------|---------|
| < 30 分钟 | 关闭或 1-2 分钟 |
| 30-60 分钟 | 2-5 分钟（推荐） |
| 1-2 小时 | 3-8 分钟 |
| > 2 小时 | 5-10 分钟 |

## 技术实现

### 新增文件

- `BREAK_FEATURE.md` - 随机间断功能详细说明文档

### 代码改动

#### 1. Bot 类新增参数

```python
def __init__(
    s,
    # ... 原有参数 ...
    enable_break=False,      # 是否启用随机间断
    break_min=2,             # 间断最小时间（分钟）
    break_max=5,             # 间断最大时间（分钟）
):
    # ... 原有代码 ...
    s.enable_break = enable_break
    s.break_min = break_min
    s.break_max = break_max
    s.last_break_time = None         # 记录上次间断时间
    s.next_break_interval = None     # 下次间断间隔
```

#### 2. 新增方法

- `_schedule_next_break()` - 计算下次间断时间（10-20分钟随机）
- `_should_take_break()` - 检查是否应该间断
- `_take_break()` - 执行间断并显示倒计时

#### 3. 主循环集成

在浏览板块的主循环中，每次循环前检查是否需要间断：

```python
while s.run:
    for cat in enabled:
        # 检查是否需要间断
        if s._should_take_break():
            s._take_break()
        
        # 继续浏览...
```

#### 4. GUI 界面更新

- 新增复选框：`s.enable_break_var`
- 新增输入框：`s.break_min_var` 和 `s.break_max_var`
- 参数验证：确保输入为有效数字且范围合理

## 已知限制

1. **首次间断延迟**：程序启动后 10-20 分钟才会触发第一次休息
2. **时间模式限制**：休息时间不计入总时间，实际运行会超过设定值
3. **停止响应**：休息期间可以立即停止，但休息倒计时会终止
