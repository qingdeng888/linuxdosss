# 快速浏览层数自定义功能

## 功能说明

v8.6 升级：**快速浏览模式的层数现在可以自定义输入了**！

之前固定为 3-5 层，现在你可以根据需要自由设置。

## 界面位置

在"浏览模式"区域：

```
⦿ 快速浏览 (爬 [3] - [5] 层换帖)
```

- **最小层数**：每个帖子至少爬多少层
- **最大层数**：每个帖子最多爬多少层
- **实际层数**：每次随机在这个范围内选择

## 参数说明

### 有效范围

- **最小值**：≥ 1（至少爬 1 层）
- **最大值**：无限制（但建议不超过 50）
- **关系**：最小值 ≤ 最大值

### 参数验证

程序会自动检查：
- ✅ 输入必须是整数
- ✅ 最小值必须 ≥ 1
- ✅ 最小值不能大于最大值
- ❌ 非法输入会弹出错误提示

## 技术实现

### 代码改动

#### 1. GUI 界面

```python
# 原来：
tk.Radiobutton(text="快速浏览（3-5层换帖）")

# 现在：
tk.Radiobutton(text="快速浏览")
tk.Label(text="(爬")
tk.Entry(textvariable=s.quick_floors_min_var, width=4)  # 最小层数
tk.Label(text="-")
tk.Entry(textvariable=s.quick_floors_max_var, width=4)  # 最大层数
tk.Label(text="层换帖)")
```

#### 2. Bot 类参数

```python
def __init__(
    s,
    # ... 原有参数 ...
    quick_floors_min=3,  # 新增
    quick_floors_max=5,  # 新增
):
    s.quick_floors_min = quick_floors_min
    s.quick_floors_max = quick_floors_max
```

#### 3. 快速浏览逻辑

```python
# 原来：
target_climb = random.randint(3, 5)  # 固定 3-5 层

# 现在：
target_climb = random.randint(s.quick_floors_min, s.quick_floors_max)  # 自定义
```

## 常见问题

**Q: 最小层数可以设置为 0 吗？**
A: 不可以，最小值必须 ≥ 1。至少要爬 1 层才有意义。

**Q: 最大层数有上限吗？**
A: 理论上没有，但建议不超过 50。层数过多会接近深度爬楼模式。

**Q: 可以只设置一个固定值吗？**
A: 可以，将最小值和最大值设为相同即可。例如都设为 5，每次都爬 5 层。

**Q: 设置 1-1 层有意义吗？**
A: 有意义。每个帖子只爬 1 层，速度最快，适合极速刷浏览话题数。

**Q: 层数越多越好吗？**
A: 不是。层数多了接近深度爬楼，失去了快速浏览的优势。建议根据需求选择。

**Q: 如何选择合适的层数？**
A: 
- 追求速度：1-3 层
- 平衡模式：3-5 层（推荐）
- 更真实：5-10 层
